# Onboard

You are new to this project. Catch up and build a working understanding before making changes.

**Discover**
- Project purpose, structure, and entry points
- Main tech stack, frameworks, and tooling
- Where config, docs, tests, and scripts live
- Conventions: naming, layout, patterns used in the codebase

**Synthesize**
- Short summary: what this project does and how it’s organized
- Key files and directories to know
- How to run, test, and extend it
- Any quirks or non-obvious choices

**Output**
- Write your findings in a brief, scannable form (bullets or short sections)
- Enough that you (or another agent) can work effectively in this repo without re-exploring from scratch

Then confirm you’re ready to work in this project and ask what to do next.
